# Идеи

Идея без гипотезы и способа проверки — шум в бэклоге. Здесь храните кандидатов и оценивайте их так, чтобы решать: **проверить сейчас / отложить / отбросить** (Lean Startup + приоритизация влияния).

Связывайте сильные идеи с [задачами](../Задачи--c110a4ad.md) и [целями](../Цели--864823f3.md). Разговоры с рынком — по правилам Mom Test (жизнь клиента и прошлое поведение, не «купили бы вы…»).

**Как оценивать:**

| Поле | Смысл |
| --- | --- |
| Доверие | Насколько вы опираетесь на факты/интервью, а не на желаемое (0 = догадка) |
| Выполняемость | Насколько дёшево проверить или сделать срез (0 = очень дорого/долго) |
| Влияние | Насколько сдвигнет ценность для клиента или ключевую метрику (0 = почти никак) |
| Баллы | Ориентир приоритета; высокий балл ≠ «делать сразу», если нет гипотезы |

Для каждой идеи кратко зафиксируйте: *гипотеза → как проверим за N дней → что будет сигналом успеха*.
