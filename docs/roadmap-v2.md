## 2. Принцип v2

> Сначала **честный сигнал и ясная ценность**, потом операционный ритм обучения, потом скорость запусков, потом полировка языка.

Порядок конфликтов моделей — как в `methodology.md`: ценность + сигнал клиента → скорость обучения → полнота процесса.

Не трогаем без отдельного запроса: лендинг, маркетинговую стратегию Prod.Swaim, массовую чистку «Библиотеки знаний».

### E — Mom Test: клиенты, интервью, отзывы

**Опора:** Fitzpatrick  
**Зачем первым:** без сигнала рынка гипотезы и MVP декоративны.

**Цель:** заменить демографический persona-подход на исследование прошлого поведения и обязательств; отзывы — не «звёзды + похвала».

**Критерии готовности:**
- [ ] В `Бизнес/Клиенты--ee4c1042.md`: зачем раздел, чеклист хороших/плохих вопросов, сильный vs слабый сигнал, red flags («купили бы?», комплименты).

### F — Lean activity: идеи → задачи → цели → рефлексия

**Опора:** Ries + Cagan  
**Зачем:** треки обещают WIP и обучение; CSV этого не держат.

**Цель:** схемы Activity поддерживают гипотезу, эксперимент, outcome и learn-цикл.

**Критерии готовности:**
- [ ] Идеи (CSV + md): гипотеза, способ проверки, статус To learn / Doing / Learned (или эквивалент), сигнал успеха, вывод, next step. Score Impact/Ease может остаться как вторичный приоритезатор.
- [ ] Задачи: тип (исследование / эксперимент / поставка / долг), связь с идеей/целью, статус; WIP-лимит описан и отражён во view-рекомендации для Notion.
- [ ] Цели: outcome (клиентский/бизнес), baseline → target, leading indicator, критерий продолжить / изменить / остановить — не только SMART+difficulty.
- [ ] Рефлексия: Build → Measure → Learn; strong/weak evidence; pivot/persevere; следующий эксперимент.
- [ ] Демо-мусор (`Goal example`, `New reflection` как пустышки) → примеры в `examples/` или нейтральные шаблоны на RU.

**Ключевые пути:**
- `Активность/Идеи--8e1bd74e.md` + `Идеи/Идеи--80dac642.csv`

### G — MVP / оффер / must-have до ценного среза

**Опора:** Olsen + владелец + Cagan  
**Зачем:** MVP = полезный (и по возможности платящий) срез; сейчас «Необходимые шаги» тянут к логотипу/баннерам до ясности ценности.

**Цель:** продукт, оффер и must-have выровнены на узкий рабочий срез, а не feature dump.

**Критерии готовности:**
- [ ] «Твой продукт»: outcome для клиента; главная гипотеза ценности; минимальный полезный/платящий срез; как поймём успех; риск (valuable / usable / feasible / viable).
- [ ] «Продукты / услуги»: очищен от ответов автора; структура оффера (для кого, проблема, что входит/не входит, цена, статус проверки); пример отдельно.
- [ ] CSV продуктов: сегмент, проблема, outcome, scope in/out, evidence / paid signal.
- [ ] «Необходимые шаги»: must-have до полезного среза / nice-to-have / не сейчас; какой риск закрывает; связь с MVP. Убрать или понизить «логотип/баннеры» как обязательный старт.
- [ ] Явно: smoke-test ≠ MVP (эксперимент допустим, продукт — нет).

**Ключевые пути:**
- `Бизнес/Твой продукт--5ac1cda1.md`
- `Бизнес/Продукты Услуги--*`
- `Бизнес/Необходимые шаги--*`
- `examples/` при необходимости

### H — Sprint / learning launches: календарь + кампании

**Опора:** Knapp + Ries  
**Зачем:** скорость = скорость обучения с людьми; нужен ритуал prototype → test.

**Цель:** календарь и планировщик кампаний поддерживают таймбокс «вопрос → прототип → тест → next steps».

**Критерии готовности:**
- [ ] Календарь: типы событий (интервью, тест прототипа, learning launch); убрать `лоловп` и шум.
- [ ] Кампании: гипотеза, аудитория, вопрос спринта, артефакт для теста, метрика/сигнал, вывод — не только каналы и купоны.
- [ ] Короткая инструкция «спринт соло-основателя» (адаптация 5 дней, без корпоративной бюрократии) — в треке или отдельной странице раздела, без ссылок на GitHub/`docs/`.
- [ ] Демо «купон −50%» либо в examples, либо переписано под learning launch.

**Ключевые пути:**
- `Активность/Календарь--*`
- `Маркетинг/Планировщик кампаний--*`
