## Что это за продукт

**Prod.Swaim / Small Business Space v.1.0 [RU]** — шаблон (digital product kit) для управления и развития цифровых продуктов / малого бизнеса.

Ценность для пользователя:
- готовые решения «не искать и не думать с нуля» под типовые ситуации;
- можно работать без ИИ или **в паре с ИИ**, проверяя и усиливая навыки;
- основной UX для людей сейчас в **Notion**; Git — рабочая копия для правок с агентом / Issues / (опционально) Obsidian.

Публичный Notion (чтение):  
https://swaim.notion.site/Small-Business-Space-v-1-0-RU-22a207cad37981ddbd72d4d9f0ed54c3

Репозиторий: https://github.com/vikiswaimer/Prod.Swaim.com

## Source of truth

| Слой | Роль |
|------|------|
| Notion (оригинал) | UI для пользователей, визуальные DB/views/формулы |
| `content/small-business-space-ru/` | Рабочая копия для ИИ и Issues (markdown + csv) |

Агент **не пишет в Notion напрямую** (нет Integration в бесплатном контуре). После merge правок в Git — чеклист «перенести в Notion» вручную владельцу.

## Как работать по задачам

1. Существенное изменение → **отдельный GitHub Issue** (цель + критерии готовности + пути файлов).
2. Правки в `content/**` (и при необходимости `docs/**`).
3. Ветки: по умолчанию `cursor/<short-name>-ffef` + draft PR. Если пользователь явно сказал «можно в main» — можно коммитить в `main`.
4. В issue/PR указывать: что перенести в Notion вручную (особенно DB schema / views / формулы).
5. Не трогать лендинг и маркетинговую стратегию, пока нет отдельного issue/чата.

## Ограничения экспорта Notion → Git

- Нет галерей, board, calendar UI, сложных linked views.
- Базы = CSV (+ md страницы записей).
- Имена файлов **укорочены** (лимит пути ОС); суффикс `--xxxxxxxx` = короткий hash id Notion.
- Внутренние ссылки уже чинились (см. `00-index.md` / `INDEX.md`). При переименованиях — обновлять ссылки.
- Карта старых→новых путей: `docs/notion-export-path-map-full.json`.

Повторный импорт из zip затирает правки в `content/` — делать только осознанно и в отдельной ветке.

## Известные логические проблемы (бэклог)

Подробности: [`docs/agent-context.md`](docs/agent-context.md).

Пакеты (порядок по умолчанию A→B→C→D):

- **A** — позиционирование продукта + очистка личных ответов автора из шаблона + Instructions.
- **B** — развести треки «Старт своего бизнеса» vs «Развитие своего бизнеса».
- **C** — язык/термины (Брэнд/Бренд, ты/вы, RU/EN), дубль «Маркетинг» (стратегия vs операции).
- **D** — пароли в шаблоне, налог 30% зашит, Untitled/демо-шум, обрыв Instructions.

## Быстрый старт агента в новом чате

1. Прочитать `AGENTS.md` + `docs/agent-context.md`.
2. Открыть активный GitHub Issue (если есть) или спросить номер.
3. Смотреть затронутые файлы через `00-index.md` / `INDEX.md`, не через «сырой» zip.
4. Делать минимальный diff под issue; не рефакторить всю библиотеку знаний без запроса.
5. В конце: commit/push, обновить issue, кратко по-русски что сделано и что перенести в Notion.
