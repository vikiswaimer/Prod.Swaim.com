# Title

```
Пакет G: MVP / оффер / must-have до ценного среза
```

---

## Цель

Выровнять «Твой продукт», оффер (Продукты / услуги) и «Необходимые шаги» на определение владельца: **MVP = уже полезный и (по возможности) платящий рабочий срез**, а не макет «для галочки» и не список «логотип + баннеры + Buffer» до ясности ценности.

Явно отделить **эксперимент / smoke-test** от **MVP-продукта**.

## Методология

**Olsen + уточнение владельца** — MVP как ценный срез.  
**Cagan** — valuable / usable / feasible / viable; outcomes.  
**Ries** — гипотеза ценности до раздувания delivery.

План: `docs/roadmap-v2.md`. Лучше после **E** (сигнал) и желательно после/рядом с **F** (гипотезы в Activity).
