# Title

```
Пакет E: Mom Test — клиенты, интервью, отзывы
```

---

## Цель

Заменить демографический persona-подход на исследование **прошлого поведения и обязательств** клиента. Отзывы вести как фиксацию сигналов ценности, а не «звёзды + похвала».

Без честного сигнала от рынка пакеты F–G (гипотезы, MVP) останутся декоративными.

## Методология

**Mom Test (Fitzpatrick)** — интервью про жизнь клиента, не про нашу идею; похвала ≠ валидация; сильный сигнал = время / деньги / intro / повтор.

См. карту в `docs/methodology.md` → модули Клиенты / Отзывы. План: `docs/roadmap-v2.md`.

## Критерии готовности

- [ ] `Бизнес/Клиенты--ee4c1042.md`: зачем раздел; чеклист хороших и плохих вопросов; сильный vs слабый сигнал; red flags («купили бы?», комплименты без обязательств).
- [ ] Шаблон записи клиента / интервью: контекст жизни и работы, workaround, частота и цена боли, что уже пробовали, обязательства (время / деньги / intro). Демография (возраст, пол) — вторична или опциональна.
- [ ] CSV-схема `Протрет пользователя` переосмыслена **или** рядом описана/добавлена логика базы «Интервью» (что именно — зафиксировать в отчёте пакета).
- [ ] `Other/Отзывы`: контекст использования, до/после, оплата или повтор, конкретная цитата, тип сигнала (weak / strong).
- [ ] Пример заполнения в `content/.../examples/` (не в основном шаблоне).
- [ ] Тон: «вы», RU; без ссылок на GitHub / `docs/` в пользовательских страницах.
- [ ] Отчёт `docs/reports/pack-e.md` + список переноса в Notion.
