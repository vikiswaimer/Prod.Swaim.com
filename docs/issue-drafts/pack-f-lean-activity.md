# Title

```
Пакет F: Lean Activity — идеи, задачи, цели, рефлексия
```

---

## Цель

Схемы и тексты модуля **Активность** должны поддерживать цикл обучения: гипотеза → проверка → вывод → следующее решение. Сейчас тексты Идей/Задач после пакета C обещают WIP и обучение, а CSV и Цели/Рефлексия — ещё старый каркас (Score / SMART / generic reflection).

## Методология

**Ries (Lean Startup)** — Build → Measure → Learn; WIP; статус проверки гипотез.  
**Cagan** — цели как **outcomes**, не список фич.

План: `docs/roadmap-v2.md`. Желательно после **E** (чтобы задачи/идеи ссылались на честный сигнал), но Git-правки возможны параллельно.

## Критерии готовности

- [ ] **Идеи** (md + CSV): гипотеза; способ проверки; статус To learn / Doing / Learned (или эквивалент на RU); сигнал успеха; вывод; next step. Impact/Ease/Score — вторичный приоритезатор, не замена гипотезы.
- [ ] **Задачи** (md + CSV): тип (исследование / эксперимент / поставка / долг); связь с идеей или целью; статус; WIP-лимит в тексте + рекомендация view для Notion.
- [ ] **Цели** (md + CSV): клиентский или бизнес-outcome; baseline → target; leading indicator; критерий продолжить / изменить / остановить. Не только SMART + difficulty/value.
- [ ] **Рефлексия**: шаблон Build → Measure → Learn; strong/weak evidence; pivot / persevere; следующий эксперимент.
- [ ] Демо-пустышки (`Goal example`, `New reflection` и т.п.) → `examples/` или нейтральные RU-шаблоны.
- [ ] Отчёт `docs/reports/pack-f.md` + перенос в Notion (свойства, формулы Score если меняются, views).
