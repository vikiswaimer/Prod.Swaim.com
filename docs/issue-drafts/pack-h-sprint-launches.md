# Title

```
Пакет H: Sprint / learning launches — календарь и кампании
```

---

## Цель

Дать соло-основателю рабочий ритуал **вопрос → прототип → тест с людьми → next steps** в Календаре и Планировщике кампаний. Сейчас Knapp-логика есть в треках, но операционные базы — старые (шум вроде `лоловп`, кампании как каналы/купоны).

## Методология

**Knapp (Sprint)** — скорость = скорость обучения; таймбокс; один чёткий вопрос.  
**Ries** — гипотеза, мера, вывод в каждом learning launch.

План: `docs/roadmap-v2.md`. Логично после **F** (есть куда писать выводы) и **G** (есть что тестировать как срез).

## Критерии готовности

- [ ] `Календарь`: типы событий — интервью, тест прототипа, learning launch (и обычные рабочие, если нужны); убрать `лоловп` и экспортный шум; демо → examples или нейтральные RU-записи.
- [ ] `Планировщик кампаний`: гипотеза; аудитория; вопрос спринта; артефакт для теста; метрика/сигнал; вывод. Не только канал и скидка.
- [ ] Короткая инструкция «спринт соло-основателя» (адаптация духа 5 дней, без корпоративной бюрократии) — в разделе Календарь/Маркетинг или якорь из трека Развитие; **без** ссылок на GitHub/`docs/` в пользовательском тексте.
- [ ] Демо «купон / −50%» — в examples или переписано под learning launch.
- [ ] Отчёт `docs/reports/pack-h.md` + Notion-чеклист (свойства, calendar/board views).
