# Обратный импорт в Notion (Git → Notion + картинки)

Цель: загрузить **проработанную** копию из `content/small-business-space-ru/` обратно в Notion **с теми же локальными картинками**, что были в экспорте.

Автозапись через Notion Integration в этом контуре **нет** — импорт делается вручную через UI Notion (ZIP).

## Что уже готово

| Файл | Назначение |
|------|------------|
| [`import/ProdSwaim-Notion-import-mini-Instructions.zip`](../import/ProdSwaim-Notion-import-mini-Instructions.zip) | **Проба**: только Instructions + 6 картинок |
| [`import/ProdSwaim-Notion-import.zip`](../import/ProdSwaim-Notion-import.zip) | Полный пакет (~6.7 MB): md + csv + 95 ассетов |
| [`scripts/pack-notion-import.py`](../scripts/pack-notion-import.py) | Пересборка ZIP из актуального `content/` |

В пакете:
- убраны суффиксы `--xxxxxxxx` из имён страниц/CSV (чтобы заголовки в Notion были читаемыми);
- относительные ссылки и пути к картинкам обновлены;
- `Instructions/_index.md` → `Instructions.md` + папка `Instructions/` с иконками (как в экспорте Notion: Page.md + Page/assets);
- Git-навигация `00-index.md` / `INDEX.md` **не** включена (шум и битые ссылки на `docs/`).

Проверка перед упаковкой: **95/95** локальных ссылок на картинки резолвятся.

## Как пробовать (рекомендуемый порядок)

### 1) Мини-проба (обязательно)

1. В Notion откройте **пустое** тестовое пространство или страницу «черновик импорта» (не публичный шаблон).
2. `Settings` → `Import` → **ZIP** (или Text & Markdown / ZIP — в зависимости от UI).
3. Загрузите `ProdSwaim-Notion-import-mini-Instructions.zip`.
4. Откройте страницу **Instructions** и проверьте:
   - иконки paper / Thunder / Folder / archive;
   - скрины Group 415 / Group 416.

Если картинок нет — почти всегда ZIP собрали «файлами без корневой папки» или импортировали только `.md` без ассетов. Пересоберите скриптом и импортируйте именно **ZIP**.

### 2) Полный импорт

1. Тот же тестовый workspace (или новый).
2. Импортируйте `ProdSwaim-Notion-import.zip`.
3. Пройдитесь выборочно:
   - `Бизнес/Твой продукт` (проработанный шаблон);
   - `Быстрый старт` (треки Старт / Развитие);
   - `Библиотека знаний/Инструменты/Tools/*` — у карточек инструментов должны быть `Untitled.png`;
   - `Брэнд/Style Guide` — цвета и typography-скрин.

## Ограничения (важно)

Импорт Markdown & CSV — **не полный клон** Notion-пространства:

| Сохранится | Скорее потеряется / упростится |
|------------|--------------------------------|
| Текст страниц после правок в Git | Board / gallery / calendar **views** |
| Локальные картинки (относительные пути) | Сложные **формулы** и rollup’ы как в оригинале |
| CSV → базы (поля упрощённо) | Linked databases / relations как в UI |
| Иерархия папок ≈ структура | Кнопки шаблонов записей, covers, иконки страниц |
| | HTML `<aside>` может выглядеть иначе |

Картинки по **внешним URL** (часть Tips) подтянутся только если Notion сможет скачать URL; локальные 95 ассетов едут в ZIP.

Модуль **Other/Пароли** попадает в полный ZIP — не публикуйте импорт «как есть» наружу.

Повторный импорт **не мержит** существующие страницы: получите новое дерево рядом. Публичный [Small Business Space](https://swaim.notion.site/Small-Business-Space-v-1-0-RU-22a207cad37981ddbd72d4d9f0ed54c3) лучше не затирать, пока мини-проба не ок.

## Пересборка после правок в Git

```bash
python3 scripts/pack-notion-import.py --also-mini --out-dir import
```

Опция `--include-git-nav` добавит `00-index.md` и `INDEX.md` (обычно не нужно для Notion).

## Дальше (если мини-проба ок)

1. Полный импорт в приватный workspace.
2. Сверить 3–5 ключевых страниц с Git.
3. Решить: дублировать в продающий шаблон вручную (copy/paste критичных страниц) **или** завести Notion Integration и скрипт точечного обновления (отдельный issue).
