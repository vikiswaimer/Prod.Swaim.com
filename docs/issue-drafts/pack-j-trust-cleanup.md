# Title

```
Пакет J: Доверие — INDEX, демо, точечный язык
```

---

## Цель

Убрать экспортный шум и остаточный язык, которые подрывают доверие к шаблону на ключевых входах: INDEX, foundation-страницы, демо-записи в оглавлении. Это не «методологический» пакет, а гигиена после A–D и параллельно с E–I.

## Методология

Качество шаблона / обещание positioning: готовые решения без ощущения «сырого экспорта». Не противоречить `methodology.md`; не раздувать scope до всей библиотеки знаний.

План: `docs/roadmap-v2.md`. Можно **частично параллелить после E**.

## Критерии готовности

- [ ] `INDEX.md`: порядок треков как в `00-index.md`; «Стратегия маркетинга» вместо голого «Маркетинг» в блоке Бизнес; без `Untitled` / `Goal example` / `лоловп` в оглавлении (или явная пометка «пример»).
- [ ] Точечная замена «малый бизнес» → продукт / ниша / цифровой продукт на foundation-страницах (Instructions, Клиенты, must-have, Позиционирование и т.п. — список в отчёте). **Не** вся «Библиотека знаний».
- [ ] EN leftovers в шаблонах записей ключевых модулей → RU; опечатки в затронутых файлах (`Протрет` → Портрет при rename с обновлением ссылок; `Momment`, `Difficulity` и т.п.).
- [ ] Решение владельца по папке `Брэнд` vs «Бренд»: (A) оставить путь Notion-hash или (B) rename + обновить `00-index` / `INDEX` / ссылки. Зафиксировать выбор в issue/отчёте.
- [ ] Прогон скрипта битых ссылок из `docs/agent-context.md` — `bad == 0` на затронутом дереве (или список оставшихся с причиной).
- [ ] Отчёт `docs/reports/pack-j.md`.
