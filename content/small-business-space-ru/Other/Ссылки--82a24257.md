# Ссылки

Раздел про **кураторскую полку рабочих ссылок**, а не свалку «открыть когда-нибудь». Здесь — зачем держать ссылки рядом с продуктом, как они ускоряют решения, как организовать и не потерять важное.

<aside>
Если ссылка не помогает принять решение, сделать эксперимент или объяснить продукт — ей, скорее всего, не место в этой базе.
</aside>

---

## Зачем это в шаблоне продукта

Основатель постоянно прыгает между статьями, доками конкурентов, инструментами, исследованиями и своими же гайдами. Без одного места ссылки живут в браузере, чатах и «где-то в Notion» — и в нужный момент их нет.

Исход раздела: за минуту находите то, чем реально пользуетесь в цикле продукта (discovery, delivery, маркетинг, поддержка).

## Что даёт бизнесу

| Даёт | Как проявляется |
|------|------------------|
| Скорость | Не ищете «ту статью про CAC» полчаса |
| Общая память | Подрядчик или будущий вы видят те же источники |
| Лучшие решения | Рядом — ресёрч, альтернативы, чужие разборы ошибок |
| Меньше шума | В закладках браузера — хаос; здесь — отобранное |
| Связь с обучением | Ссылка подкрепляет гипотезу, интервью, метрику |

Ссылка сама по себе не стратегия. Она — топливо для мысли и быстрый доступ к материалу.

---

## Что класть (и чего не класть)

**Класть**
- методики и фреймворки, которыми пользуетесь (ICE, JTBD, гайды по метрикам);
- конкуренты, альтернативы, обзоры рынка;
- ваши публичные материалы и канонические доки;
- инструменты стека (документация, биллинг, аналитика);
- исследования и интервью-источники, к которым возвращаетесь.

**Не класть**
- разовые вкладки «потом почитаю» без пометки зачем;
- протухшие URL (лучше удалить или пометить «мертво»);
- секретные админки и URL с токенами в query — это риск (см. «Цифровая безопасность»);
- дубли одного и того же под разными названиями.

## Как организовать

Простая схема для соло:

1. **Название** — понятное вам через полгода.
2. **URL** — канонический, без трекинг-мусора, если можно.
3. **Зачем** — одно предложение: «приоритизация идей», «считаем CAC».
4. **Тема / тег** — продукт, маркетинг, метрики, инструменты, конкуренты, юридическое…
5. **Статус** — активно / архив / проверить.

Раз в квартал: пройтись, удалить мёртвое, обновить «зачем».

Демо-записи в таблице — примеры тем; замените своим рабочим набором.

## Инструменты (примеры)

| Задача | Примеры |
|--------|---------|
| Канон рядом с продуктом | эта база в шаблоне |
| Быстрый захват из браузера | Notion Web Clipper, Raindrop, Pocket, закладки с тегами |
| Команда | общая DB + правило «одна ссылка — одно зачем» |
| Чтение офлайн / later | Readwise, Instapaper — но в шаблон переносите только отобранное |

Не плодите три системы «правды». Выберите: либо эта база — канон, либо внешний менеджер закладок, а здесь — только топ рабочий.

## Практический мини-чеклист

- [ ] В базе только ссылки, к которым возвращались хотя бы раз по делу
- [ ] У каждой есть «зачем», не только URL
- [ ] Нет URL с секретами и персональными данными в адресе
- [ ] Есть 5–15 ссылок под текущий цикл (не 300 «на всякий»)
- [ ] Раз в квартал чистка: удалить / архив / обновить
- [ ] Важные методики из ссылок при необходимости пересказаны своими словами в заметках (ссылка может исчезнуть)

## Что делать с этим разделом

- Перенесите сюда 10 ссылок, которыми уже пользуетесь — не идеальный каталог интернета.
- Рядом с идеей или задачей можно держать 1–2 опорных URL из этой базы.
- Если ссылка устарела — удаляйте без жалости: ценность в актуальности, не в объёме.
